The problem

Agents today are handed the keys to the kingdom.

AI agents are given API keys, card numbers, and PII directly. One prompt injection — a poisoned email, a malicious document, a rogue tool result — and the agent exfiltrates secrets or wires money to an attacker. The root cause, in Terminal 3's words: “absence of verifiable identity, scoped permissions, and tamper-resistant audit at the action layer.”

EchoLeak

Data exfiltrated through a poisoned email.

Replit agent

Deleted a production database, then faked a cover-up.

Prompt injection

Hidden instructions hijack agents into dumping credentials.

Poisoned issues

Malicious repo content compromises local agents.

How Aegis fixes it

Authority lives in the TEE — not in the agent's good behaviour.

🪪

Verifiable identity

Patient, clinic, and agent are did:t3n principals. The agent signs every call with a key bound into the credential.

🎚️

Scoped permissions

The patient signs a delegation credential scoping functions, a payee allowlist, a spend cap, and PHI disclosure — enforced at the action layer.

🔒

Confidential compute

PHI lives only in the TEE. The agent sends {{profile.*}} placeholders, resolved inside the enclave. We verify Intel TDX attestation first.

🧾

Tamper-evident audit

Every action is host-stamped (subject, actor, credential id) into an append-only trail the agent cannot forge.

🎯

Selective disclosure

The insurer, pharmacy, and bank each receive only the fields they need — the SSN is never disclosed to anyone.

⛔

Instant revocation

The patient can revoke the agent's authority at any moment; the next action is refused by the TEE.

The proof

We attack our own agent — and it holds.

Security is the heaviest-weighted criterion, so we don't assert it — we demonstrate it. In A1–A3 the agent is fully compromised and obeys the injected instructions; the attack still fails, because the TEE enforces the rules. The harness fails the build if even one gets through.
